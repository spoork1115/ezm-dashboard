# 작업 지침

- 사용자가 별도로 요청하지 않는 한 모든 진행 메시지와 최종 보고는 한국어로 작성한다.
- 사용자가 "배포"라고 말하면 기본적으로 Git 저장소에 변경사항을 반영하는 것을 의미한다.
- 코드 수정 작업이 완료되면 가능한 범위에서 검증을 수행한다.
- 검증 후 변경사항을 확인하고, 배포 가능한 상태라면 Git 커밋과 원격 저장소 푸시까지 진행한다.
- Git 커밋/푸시 전에 변경 파일, 검증 결과, 커밋 메시지를 확인한다.
- 원격 저장소 푸시나 인증에 승인이 필요하면 필요한 승인 절차를 거쳐 진행한다.
- Vercel, 서버, 클라우드 같은 외부 서비스 직접 배포는 사용자가 명시적으로 요청한 경우에만 진행한다.

참고: 자동 지침 적용을 위해 같은 내용이 `AGENTS.md`에도 등록되어 있다.
